================================================================================
모듈 간 데이터 흐름 (Data Flow)
================================================================================

사용자 질문 입력 (app.py)
    ↓
[1] QuestionClassifier
    ├→ "medical" / "hospital" / "general" / "unknown" 분류
    └→ Classification 결과 반환
    ↓
[2] QueryOrchestrator (의도에 따라 라우팅)
    │
    ├─→ medical/hospital 질문
    │   ├→ LangGraphRAGPipeline 호출
    │   │   ├→ [RetrieverNode] VectorStoreRetriever.search()
    │   │   ├→ [RelevanceNode] 관련성 판단
    │   │   ├→ IF 관련문서 없음 → [WebSearchNode] 트리거
    │   │   │   ├→ HospitalWebSearcher or GeneralWebSearcher
    │   │   │   └→ 검색 결과 반환
    │   │   └→ [GenerationNode] LLM 기반 답변 생성
    │   └→ 최종 응답 + 매핑 정보
    │
    ├─→ hospital(위치) 질문
    │   ├→ HospitalMapper.extract_and_search()
    │   ├→ 병원 위치 정보 조회
    │   └→ 좌표 + 병원 정보 반환
    │
    └─→ general 질문
        └→ GeneralWebSearcher 호출
            └→ 웹 검색 결과 반환

최종 응답 (UI에 표시)


================================================================================
핵심 클래스/인터페이스
================================================================================

1. QuestionClassifier
   - intent: str ← classify(question: str)
   - 추후 LLM 기반으로 확장 가능

2. VectorStoreRetriever
   - documents: List[Document] ← search(query: str, top_k: int)
   - 추후 다양한 벡터DB 지원 가능

3. HospitalWebSearcher / GeneralWebSearcher
   - results: List[Dict] ← search(query: str)
   - Fallback 시스템으로 활용

4. HospitalMapper
   - locations: List[Dict] ← extract_and_search(query: str)
   - 좌표 + 병원 정보 반환

5. LangGraphRAGPipeline
   - response: str ← invoke(query: str, docs: List[Document])
   - 노드 기반 워크플로우로 확장 가능

6. QueryOrchestrator
   - result: Dict ← process(question: str)
   - 전체 흐름 조정

================================================================================
확장 포인트 (Hook Points for Teams)
================================================================================

1. Classifier 고도화
   - src/classifier/question_classifier.py → LLM 기반 분류기로 교체
   - intent 카테고리 추가/수정

2. Retriever 고도화
   - src/retriever/chroma_adapter.py → 다른 벡터DB (Pinecone, Weaviate 등) 연동
   - 임베딩 모델 변경 (Ollama, HuggingFace 등)

3. RAG 파이프라인 고도화
   - src/rag/nodes/ → 새로운 노드 추가
   - src/rag/graph_builder.py → 복잡한 워크플로우 구성

4. 웹 검색 고도화
   - src/web/ → 실제 API 연동 (Google, Naver, 카카오 등)

5. 병원 정보 시스템 고도화
   - src/mapping/ → 실제 카카오맵 API 연동
   - 병원 정보 데이터베이스 구축

6. UI 고도화
   - app.py → 더 복잡한 Streamlit 대시보드
   - 히스토리, 피드백, 평가 시스템 추가
